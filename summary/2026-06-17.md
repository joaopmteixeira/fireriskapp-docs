# Resumo do dia - 17 de junho de 2026

## O que foi feito hoje

- **Configuração do acesso à base de dados com 3 perfis distintos:** criados 3 utilizadores de base de dados com permissões diferentes — um com acesso total (para migrações e manutenção), um com acesso de trabalho normal (para a aplicação), e um só de leitura (para relatórios ou auditorias). Ficaram visíveis e configurados no Adminer.

- **Política de backups automáticos em 3 níveis:** o sistema de backups passou a ter 3 ritmos diferentes — todos os dias (guardando os últimos 7), de 3 em 3 dias (guardando os últimos 10, cobrindo um mês), e no primeiro dia de cada mês (guardado para sempre). Útil para recuperar dados de qualquer ponto no tempo, incluindo para efeitos de dissertação.

- **Escolha e limpeza do gestor de base de dados:** após testar as duas ferramentas em staging (pgAdmin e Adminer), foi escolhido o Adminer por ser mais simples e direto. O pgAdmin foi completamente removido do sistema (sem deixar rastros). O Adminer ficou acessível na porta 5050, protegido por palavra-passe.

- **Recuperação de conta apagada acidentalmente:** o utilizador JoaoTeixeira foi apagado por engano no Adminer e recuperado a partir do backup automático do próprio dia, sem perda de dados.

- **Diagnóstico e resolução de migração que não aplicava:** a migração de base de dados com os novos perfis não estava a ser aplicada porque a imagem Docker tinha código antigo. Após reconstruir a imagem, tudo ficou operacional.

## Estado do projeto

A infra de staging está completa e estável: a aplicação corre numa VM local (Proxmox), acessível publicamente via `https://chichorro.joaopmteixeira.net`, com base de dados PostgreSQL local, backups automáticos diferenciados, 3 perfis de acesso à BD, e o Adminer como ferramenta de gestão. O pgAdmin foi avaliado e descartado. O próximo passo natural é o hardening da configuração Docker (SEC-13) e depois a monitorização self-hosted (INFRA-08).
