# Resumo do dia - 18 de junho de 2026

## O que foi feito hoje

- **Proteção contra secrets no código**: adicionámos um mecanismo automático que impede que palavras-passe e chaves de API sejam acidentalmente incluídas no código — o sistema de CI agora bloqueia qualquer commit que contenha segredos.
- **Isolamento da base de dados**: a base de dados da aplicação passou a estar numa rede Docker interna, inacessível diretamente do exterior; apenas os serviços que precisam dela têm acesso.
- **Arranque automático da aplicação**: a aplicação na VM agora arranca automaticamente quando o servidor é reiniciado, sem necessidade de intervenção manual.
- **Alertas de monitorização por email**: configurámos três alertas automáticos que enviam email para `chichorrofireriskapp@gmail.com`: (1) se o site ficar em baixo, (2) se o disco da VM encher acima de 80%, (3) se o backup diário da base de dados não tiver sido feito nas últimas 26 horas.
- **Acesso SSH via Cloudflare tentado e revertido**: tentámos configurar acesso remoto ao servidor sem VPN, mas a solução era incompatível com o plano gratuito do Cloudflare e adicionava fricção sem benefício real — decisão de reverter.

## Estado do projeto
