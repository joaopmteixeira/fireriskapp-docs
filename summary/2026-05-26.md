# Resumo do dia - 26 de maio de 2026

## O que foi feito hoje

- **Corrigida uma falha de segurança grave:** qualquer pessoa com uma conta na aplicação conseguia ver os e-mails, IPs e dados de todos os outros utilizadores nas páginas de administração. Foi implementado um sistema de roles que garante que apenas administradores acedem a essas páginas — qualquer outra conta recebe um erro de acesso negado.

- **Criada a secção de administração na aplicação:** os utilizadores admin passam a ver um grupo "ADMIN" no fundo do menu lateral, com acesso a duas páginas novas — uma tabela com todos os utilizadores registados e os seus roles, e uma tabela com o histórico de acessos (quem entrou, quando e a partir de que endereço IP).

- **Resolvidos 6 problemas de segurança identificados pelo Codex:** a revisão externa tinha encontrado falhas na validação de URLs, nas credenciais de base de dados, no script de backup e na política de segurança do browser. Todos foram corrigidos na branch `audit-fix-2`: a validação de URLs passou a ser mais rigorosa, o Alembic passa a exigir credencial separada em produção, o backup passou a usar uma conta com apenas permissão de leitura, e o Sentry voltou a receber relatórios de erros do browser.

- **Descoberto e corrigido o motivo pelo qual o login nunca funcionava em produção:** o Supabase tem uma funcionalidade de segurança ativa por defeito (Row Level Security) que tornava todos os registos da base de dados invisíveis. O problema foi resolvido com uma migração automática.

- **Corrigido um bug que bloqueava todos os formulários POST em produção:** o cookie de segurança CSRF não conseguia ser lido pelo frontend porque o backend e o frontend estão em domínios diferentes. Corrigido com uma configuração de domínio no middleware.

- **Documentados os passos manuais necessários no Supabase e GitHub:** para completar o isolamento de credenciais do backup é necessário criar um utilizador específico no Supabase e adicionar um novo secret ao GitHub — os comandos SQL e instruções estão documentados em `DEPLOY_PRODUCTION.md`.
