# Resumo do dia - 28 de maio de 2026

## O que foi feito hoje

- **Segurança reforçada:** removido o código antigo de verificação de palavras-passe (werkzeug) agora que todos os utilizadores já têm o formato mais seguro (argon2id). O sistema ficou mais simples e com menos dependências externas.

- **Organização da documentação:** todos os 58 ficheiros de planos de tarefa foram uniformizados com o mesmo formato de cabeçalho. Um ficheiro duplicado foi eliminado. Duas entradas incorretas no TODO foram corrigidas.

- **Correção de bug no cálculo POI:** os formulários de "Instalações de aquecimento" e "Atividade do edifício" estavam a enviar campos escondidos vazios para o servidor, causando erros HTTP 422. Corrigido em dois passos: o servidor passou a aceitar esses campos como opcionais, e o frontend passou a omiti-los quando não visíveis.

- **Sincronização bidirecional CTI↔ATIV:** o campo "Tipo de edifício" no CTI estava bloqueado sempre que o POI tinha um valor preenchido. Agora o campo é editável em ambos os lados (CTI e POI ATIV), com espelho automático entre eles ao mudar qualquer um.

- **Correção de navegação após importar sessão:** ao importar ou limpar uma sessão, as páginas não actualizavam o conteúdo visível. Corrigido forçando o React a recriar a página quando a sessão muda.

## Estado do projeto

A aplicação está em produção estável no Render (backend) e Cloudflare Pages (frontend). Os bugs de cálculo do módulo POI foram resolvidos e o fluxo de cálculo está funcional. A segurança do sistema de autenticação foi simplificada após a migração completa para argon2id. O projeto encontra-se numa fase de refinamento e correção de bugs pontuais, sem novas funcionalidades em curso.

## Próximo passo

Verificar em produção os cálculos POI IA e POI ATIV com diferentes configurações, e testar o smoke test completo (login → cálculo → resultados → exportar sessão).
