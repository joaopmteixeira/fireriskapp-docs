# Resumo do dia - 24 de maio de 2026

## O que foi feito hoje

- **Separação de utilizadores na base de dados:** foi criada a configuração técnica para que a aplicação use um utilizador com permissões limitadas (`chichorro_runtime`), enquanto as atualizações de estrutura da base de dados continuam a usar um utilizador administrador. Isto reduz o impacto caso a app seja comprometida.
- **Alertas de falha de backup:** se o backup automático de dados (que corre de 3 em 3 dias) falhar, é enviado um e-mail de alerta automaticamente para o João. Anteriormente, uma falha passaria despercebida.
- **Rastreabilidade de pedidos:** cada pedido à API recebe agora um identificador único (`X-Request-ID`). Quando ocorre um erro, este ID aparece nos logs do Sentry, facilitando encontrar a causa.
- **Remoção de código morto:** foram eliminados 4 endpoints antigos da API que não eram usados pelo frontend há muito tempo. Menos código = menos superfície de ataque.
- **Documentação de deploy atualizada:** os guias de instalação e variáveis de ambiente foram reescritos e consolidados num único documento canónico. Um ficheiro desatualizado foi apagado.
- **Merge do ciclo de segurança:** os 16 planos de segurança implementados nas últimas semanas foram integrados na branch principal de desenvolvimento (`3.1-dev`). O ciclo de audit está completo.
- **Ações manuais nos dashboards concluídas:** o João configurou o utilizador restrito na Supabase, atualizou as variáveis no Render, configurou monitores UptimeRobot e Sentry, e validou as variáveis no Cloudflare Pages.
- **Bug de deploy encontrado e corrigido:** o código que gere a estrutura da base de dados (alembic) passava uma ligação direta ao PostgreSQL quando devia usar a camada de abstração SQLAlchemy — bug invisível em desenvolvimento, só apareceu em produção. Corrigido e deploy ficou verde às 18:02 UTC.
