# Resumo do dia — 19 de maio de 2026

## O que foi feito hoje

- **Ativámos o sistema de monitorização completo da aplicação.** A partir de hoje, qualquer erro que ocorra em produção — tanto no site como no servidor — é automaticamente registado numa plataforma chamada Sentry, com gravação do que o utilizador estava a fazer quando o erro aconteceu. Também configurámos um sistema de alertas que verifica se o servidor está a responder a cada 5 minutos.

- **Criámos uma rotina automática de backup da base de dados.** De três em três dias, uma tarefa automática corre na cloud (GitHub), exporta todos os utilizadores e registos de acesso para ficheiros seguros, e guarda-os durante 90 dias. Nunca mais precisamos de fazer backups manualmente.

- **Fizemos uma auditoria detalhada à segurança do código.** Pedimos ao sistema de IA Codex para analisar todo o código e identificar riscos de segurança. Depois verificámos cada risco diretamente no código, reclassificámos a gravidade de alguns e documentámos tudo num ficheiro local (que não vai para o GitHub, pois descreve vulnerabilidades em detalhe).

- **Identificámos e registámos 11 novas tarefas de segurança.** As mais urgentes são: remover um ficheiro legado que guarda credenciais de acesso no código do site (visíveis a qualquer visitante), e adicionar verificação de administrador nas páginas de gestão (atualmente qualquer utilizador registado pode ver os dados de todos os outros).

- **Atualizámos e corrigimos a documentação do projeto.** Várias referências antigas a "Flask" foram corrigidas para "FastAPI" (o servidor atual), tarefas já concluídas foram marcadas como tal, e os novos identificadores de tarefa foram adicionados a todos os ficheiros de gestão do projeto.

## Estado do projeto

O projeto CHICHORRO 3.1 está operacional em produção com monitorização completa. Toda a infraestrutura de suporte está agora no lugar: autenticação segura, monitorização de erros, alertas de disponibilidade e backups automáticos. A auditoria de segurança realizada hoje identificou 11 tarefas pendentes de hardening — nenhuma bloqueia o funcionamento atual, mas algumas devem ser resolvidas antes de abrir a plataforma a utilizadores externos. O código do modelo de avaliação de risco de incêndio (CHICHORRO 3.1) está estável.

## Próximo passo
